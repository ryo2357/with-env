# 実装プラン: with-env CLI

このドキュメントは、アイディアファイル「01\_プロジェクト概要.md」に基づき、APIキー隠蔽・管理CLIツール `with-env` を実装するための詳細な計画です。

## 1. 目的

- AIエージェント環境下でのシークレット情報の流出リスクを低減する。
- `.env` ファイルにプレースホルダ（`WITH{{KEY}}`）を記述し、実行時のみ実数値に展開して環境変数として注入する。

## 2. 実装フェーズ

### フェーズ1: プロジェクト基盤の整備

- [ ] `Cargo.toml` に必要な依存関係を追加
  - `clap`: コマンドライン引数解析
  - `dotenvy`: `.env` ファイルのパース
  - `serde` / `serde_json`: 設定・シークレットファイルの読み込み
  - `dirs`: ホームディレクトリ（`~/.config`）のパス解決
  - `anyhow`: エラーハンドリング
  - `nix` または `std::process::Command`: コマンド実行（exec）
- [ ] 基本的なコマンド構造（`run` コマンド）の定義

### フェーズ2: 設定・シークレット管理

- [ ] `~/.config/with-env/secrets.json` の読み込みロジックの実装
- [ ] `~/.config/with-env/settings.json` の読み込みと構造体定義
  - `allow_command`: 許可ディレクトリ・コマンドのリスト
  - `allow_run_path`: 実行許可パスの制限

### フェーズ3: 環境変数展開ロジック

- [ ] `.env` ファイルの読み込みとパース
- [ ] `WITH{{KEY}}` パターンの検出
- `secrets.json` からの値の取得と変換
- [ ] 未解決のプレースホルダがある場合のエラー処理

### フェーズ4: セキュリティと実行制限

- [ ] `allow_run_path` によるカレントディレクトリのバリデーション
- [ ] `allow_command` による実行コマンドのバリデーション（`&` 等のメタ文字除外）
- [ ] 実行スコープの制限（子プロセスへの注入のみ）

### フェーズ5: コマンド実行と最終調整

- [ ] 引数を受け取って外部プロセスを起動するロジックの実装
- [ ] 詳細なエラーメッセージの実装
- [ ] テストコードの作成

## 3. ファイル構造（予定）

- `src/main.rs`: エントリポイント、CLI引数処理
- `src/config.rs`: 設定・シークレット管理
- `src/env.rs`: `.env` パース・展開
- `src/executor.rs`: コマンド実行

## 4. 成功基準

- `with-env run -- cargo run` が正常に動作し、`.env` 内のプレースホルダが展開されていること。
- 許可されていないディレクトリやコマンドで実行した際に、適切にエラー終了すること。
- シークレットがメモリ上でのみ展開され、ファイルが書き換えられないこと。
