# APIキー隠蔽・管理CLI

## 概要

- AIエージェントを使った開発にてAPIキーの流出可能性を下げたい
- .env運用を維持したまま
- シークレットを分離し
- run時のみ注入
- AIやツールから直接読みづらくする

## 要件

### runコマンド

`with-env run -- <command>`

- カレントディレクトリの .env を読む
- WITH{{KEY}} を検出
- ローカル設定からシークレット取得
- メモリ上で展開（ファイルは変更しない）
- 環境変数として注入
- <command> を exec 実行

### .env仕様

```.env
DATABASE_URL=postgres://user:WITH{{DB_PASSWORD}}@localhost/db
API_KEY=WITH{{API_KEY}}
```

- WITH{{KEY}} を展開して環境変数にキャスト
- 通常のdotenvと完全互換
- 未解決はエラーで停止

### シークレット管理

- dir::home_dir()に保存。`~/.config/with-env/secrets.json`

```json
{
  "DB_PASSWORD": "xxx",
  "API_KEY": "key-xxx"
}
```

### コマンド、実行フォルダの制限等の設定

- `~/.config/with-env/settings.json`に保存
- `allow_command`は実行できる子プロセスを定義
- `allow_command`に対して追加の引数は書けるが`&`などは使えないようにしたい
- `allow_run_path`は`with-env run`を実行できるディレクトリの制限

```json
{
  "allow_command": [
    ["cargo", "run"],
    ["cargo", "test"],
    ["cargo", "build"]
  ],
  "allow_command": ["E:/dev/project1", "E:/dev/project2"]
}
```

### 実行スコープ制限

- 環境変数は 子プロセスのみに適用
- 親プロセス・ファイルには影響しない

## 非機能要件

- 実行中プロセスからは参照可能
- secrets.jsonは平文（ローカル前提）
