# 実装報告書: CLI基盤と環境変数展開ロジックの実装

## 実装内容の概要

[01\_実装プラン.md](../plans/01_実装プラン.md) のフェーズ1からフェーズ3に相当する機能を実装しました。

1.  **CLI基盤の構築**: `clap` を使用して、外部コマンドを実行するための `run` サブコマンドを定義しました。
2.  **設定・シークレット管理**:
    - `secrets.json`, `settings.json` の構造定義と読み込みロジックを実装。
    - 開発(debug)ビルド時は `.dev_config/` ディレクトリを参照する仕組みを導入しました。
3.  **環境変数展開ロジック**:
    - `dotenvy` を使用して `.env` ファイルを読み込み、`WITH{{KEY}}` 形式のプレースホルダを `secrets.json` 内の値で置換する機能を実装しました。
    - 開発時は `.dev_config/.env` を優先的に読み込むように調整しました。

## 実装結果とコーディング計画の差異

### 1. `allow_command` のデータ構造変更

- **計画**: `Vec<String>` (コマンド名のリスト)
- **結果**: `Vec<Vec<String>>` (コマンドと引数の組み合わせのリスト)
- **理由**: `cargo run` のように「特定のサブコマンドまで含めて許可したい」という要望に対応するため、より柔軟で安全な構造に変更しました。

### 2. 開発用設定ディレクトリ名の変更

- **計画**: `~/.config/with-env` または `.env.local`
- **結果**: `.dev_config`
- **理由**: ユーザーの要望により、開発環境での設定ファイルを隔離しやすくするため `.dev_config` を標準の開発用ディレクトリとして採用しました。
